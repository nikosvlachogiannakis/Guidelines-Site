Pulling Changes and Handling Conflicts

Basic Concept

When collaborators push changes to GitHub, your local repo becomes “behind” the remote version. You'll want to:

  • Fetch their updates
  • Review what changed
  • Merge or rebase safely

All without overwriting your own uncommitted work.

Full LazyGit Workflow

1. Open LazyGit

lazygit

You will see panels for branches, files and commits.

2. Fetch Updates

Press:

f

This fetches remote updates without merging them.

3. Pull Changes

Press:

p

Select the remote and branch.

If there are no conflicts, Git merges automatically.

Handling Conflicts

Conflicted files appear in red.

Use:

  • m → Open merge menu
  • e → Edit file manually
  • d → View diff

Resolve Conflicts

  • y → Use ours
  • t → Use theirs
  • b → Use both
  • e → Manual edit

After fixing the file:

space

This stages the resolved file.

Commit the Merge

c

This commits the resolved merge.

Push Changes

Press:

P

This pushes the branch to GitHub.

Pro Tips

  • h → View commit history
  • u → Undo last action
  • x → Abort merge or rebase
  • b → Switch branches

Without LazyGit

1. Save your current work

git status
If needed: 
git add .
git commit -m "Brief explanation of what changes were made"
Or stash: 
git stash

2. Fetch remote changes

git fetch origin

3. See what changed

git diff main origin/main
To view commits: 
git log main..origin/main --oneline

4. Merge changes

git pull origin main
Equivalent to: 
git fetch origin
git merge origin/main
For cleaner history: 
git pull --rebase origin main

Pull Only Specific Commits

git fetch origin
git cherry-pick <commit-hash>

Restore Stashed Work

git stash pop

Force Reset Local Repo

If you want to completely discard local changes:

git fetch origin
git reset --hard origin/main
git clean -fd
