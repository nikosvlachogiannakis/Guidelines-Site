New GitHub Repository

Option 1. Create a repo on GitHub and clone it locally

  • Create a Repository on GitHub (either public or private) with a README.md file
  • Go to the path where you want to have your project:
cd path
  • Copy the link of the SSH key (ssh_key) from your repository in GitHub and write the following in terminal:
git clone ssh_key
  • You can open the file we cloned in VS Code and work on it.

Option 2. Create a File locally and push to GitHub

  • Go to GitHub and create a repo. Name it the same as your local repo and do NOT create a README.md (you already have one locally). You want your repo to be completely empty.
  • Once the repo is created in GitHub, go to Code → SSH and copy the repo SSH link.
  • Locally, on your cmd/bash do:
git init

git add .

git commit -m "Initial commit"

git remote add origin ssh_link-you-copied

git branch -M main

git push -u origin main

Your commits should be pushed to GitHub, and your local repo will be connected to the GitHub repo.

Option 3. Using GitHub CLI (gh)

This option uses HTTPS protocol instead of SSH.

  • Install GitHub CLI. After successful installation of gh (if you install shell repo gh will be installed too), it asks for a token. Follow the link and create a token with the proposed permissions. After this action, you can use gh to do things in GitHub.
  • Connect it to your GitHub with:
gh auth login

and follow the prompts.

To create a new public repo in the current folder, do:

git init

git add .

git commit -m "Initial commit"

gh repo create REPO_NAME --public --source=. --push

Clone a repo:

gh repo clone repo_name

See all your repos in GitHub:

gh repo list

Important Note

Token is used for gh, GitHub's CLI

If that token expires you need to run the following to make sure the token is updated.

In the remote machines' Terminal:

gh auth login

Then it will ask you:

? Where do you use GitHub?

Select:

GitHub.com

Then it will ask:

? What is your preferred protocol for Git operations on this host?

Select:

HTTPS

Question:

? Authenticate Git with your GitHub credentials?

Select:

Yes

Question:

? How would you like to authenticate GitHub CLI?

Select:

Paste an authentication token

Then on the following:

? Paste your authentication token:

Paste the token you (re-)generated and copied from GitHub.

Then it will say that you are logged in as account_name or something similar.
