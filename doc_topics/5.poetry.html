Python Poetry

Poetry is a tool that helps you manage your Python projects more easily. It installs packages, keeps track of dependencies, and ensures your project works consistently across different systems.

It also creates a clean structure for your code and simplifies sharing or publishing your project. Poetry automatically manages virtual environments and project configuration, making development easier and more organized.

Download Poetry

Download from: Poetry Documentation

Use Poetry on an Existing Project

Open Terminal and Enter the Project Directory

cd my_project

Initialize Poetry

poetry init

This command creates or updates the pyproject.toml file.

Package name [my_project]:
Version [0.1.0]:
Description []:
Author [Your Name <you@example.com>, n to skip]:
License []:
Compatible Python versions [^3.11]:
Would you like to define dependencies interactively?

Example generated configuration:

[tool.poetry]
name = "my_project"
version = "0.1.0"
description = ""
authors = ["Your Name <you@example.com>"]
license = "MIT"
readme = "README.md"
packages = [{include = "my_project"}]

[tool.poetry.dependencies]
python = "^3.11"

[build-system]
requires = ["poetry-core"]
build-backend = "poetry.core.masonry.api"

This method does not create project folders. It simply adds Poetry management to an existing project.

Create a New Project with Poetry

poetry new my_project

This generates the following structure:

my_project/
├── my_project/
│   └── __init__.py
├── tests/
│   └── test_my_project.py
├── pyproject.toml

Enter the Project Directory

cd my_project

Open in VS Code

code .

Install Dependencies

poetry install

Activate the Environment

poetry shell

Configure VS Code Interpreter

Press Ctrl + Shift + P → Select Python: Select Interpreter and choose the Poetry environment.

Add Dependencies

Add a package:

poetry add requests

Add development dependency:

poetry add --dev pytest

Virtual Environments

A virtual environment is an isolated Python workspace where project dependencies are installed separately from the system Python.

Activate Environment

poetry shell

Example prompt:

(.venv) user@machine my_project %

Run Commands Without Activating

poetry run python main.py
poetry run pytest

Exit the Environment

exit
deactivate
Ctrl + D

Virtual Environment Location

Default directories:

  • Linux/macOS: ~/.cache/pypoetry/virtualenvs/
  • Windows: %APPDATA%\pypoetry\virtualenvs\

Create Environment Inside Project

poetry config virtualenvs.in-project true

This creates:

my_project/
├── .venv/
├── pyproject.toml

Check Environment Information

poetry env info

Recreate Environment

poetry env remove python
poetry install

Typical Workflow

cd my_project
poetry shell
code .
pytest
python main.py
poetry add requests
exit
