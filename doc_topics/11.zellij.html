Zellij Binary Installation (Linux x86_64) & Poetry Workflow

Check System Architecture

uname -s
uname -m

If output shows OS: Linux and Architecture: x86_64, download:

zellij-x86_64-unknown-linux-musl.tar.gz from Zellij GitHub releases

curl -LO https://github.com/zellij-org/zellij/releases/download/v0.41.2/zellij-x86_64-unknown-linux-musl.tar.gz
tar -xvf zellij*.tar.gz
chmod +x zellij
./zellij

Optionally, move binary to local bin and add to PATH:

mkdir -p ~/.local/bin
mv zellij ~/.local/bin/
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
which zellij
zellij --version

Zellij + Poetry Workflow Cheat Sheet

1. SSH and Start Zellij

ssh user@remote-machine
zellij

2. Open Tabs or Panels

Tabs (separate shell per script)

  • New tab: Ctrl+t → n
  • Next tab: Ctrl+t → Tab
  • Previous tab: Ctrl+g → Shift+Tab
  • Go to tab by number: Ctrl+g → 0-9

Panels (split view in same tab)

  • Split vertically: Ctrl+g → v
  • Split horizontally: Ctrl+g → h
  • Move between panels: Ctrl+g → arrow keys
  • Resize panels: Ctrl+g → hold Alt + arrow keys

3. Navigate to Poetry Project

cd ~/my_project

4. Activate Poetry Environment

Option A — Interactive Shell

poetry shell
python script1.py

Repeat in another tab/panel:

poetry shell
python script2.py

Option B — Direct Run (non-interactive)

poetry run python script1.py
poetry run python script2.py

5. Detach and Reattach Sessions

  • Detach: Ctrl+g → d
  • List sessions: 
    zellij list-sessions
  • Reattach session: 
    zellij attach -s session_name

6. Optional Tips

  • Name tabs after scripts: Ctrl+g → n
  • Close tab/panel: Ctrl+g → x
  • Combine panels + tabs to run multiple scripts efficiently

7. Example Layouts

Tabs:

  • Tab 1: script1.py (poetry run)
  • Tab 2: script2.py (poetry run)
  • Tab 3: monitoring/logs (optional)

Panels (one tab):

  • Left panel: script1.py
  • Right panel: script2.py

Key Takeaways

  • Use tabs for separate scripts, panels for simultaneous monitoring
  • Always run scripts inside the Poetry environment
  • Detach sessions to keep scripts running after SSH disconnect
  • Name tabs/panels for clarity
