Remote Machine Connection (With SSH Keys)

In order to connect to a remote machine you can do it in two ways:

Option 1: With Password

ssh user@remote-server

Then you must enter the password of the remote machine and you are all set!

Note: The first time you connect, SSH may ask you to confirm the server's authenticity — type yes and press Enter.

Option 2: Without Password

1. Generate an SSH key (If you don't have one)

Locally, on your computer, open Terminal (if you have Windows open Git Bash) and run: 

ssh-keygen -t ed25519 -C "your_email@example.com" -f ~/.ssh/name_of_the_key
  • Press Enter to accept the default path: C:\Users\username\.ssh\...
  • Choose no passphrase to enable password-less login

2. Copy the Public Key to the Remote Server

ssh-copy-id user@remote-server

3. Use SSH Config File (Optional in order to have a simpler name for the connection)

Create or edit:

~/.ssh/config

Add:

Host myserver
HostName REMOTE_SERVER
User USERNAME
IdentityFile ~/.ssh/id_ed25519

Then press Ctrl+O and Ctrl+Enter to save and exit.

Now you can simply run:

ssh myserver

4. Test the Password-less SSH Login

Run:

ssh user@remote-server

OR if you did Step 3 you can run:

ssh myserver

You should be logged in successfully!

Log Out From The Remote Machine

Ctrl D

OR

exit
