Docker Guide

Docker is a platform that allows you to package applications into containers, ensuring consistency across environments. Containers are lightweight, isolated environments that share the host OS kernel.

More about Docker

Install Docker

Install Docker Desktop (Windows/macOS) or Docker Engine (Linux) and verify installation by running:

docker --version

This prints the installed Docker version.

Hello World Quick Test

Run a simple test container to confirm Docker works:

docker run hello-world

This command pulls the "hello-world" image and runs it.

Basic Concepts

  • Image: Read-only template for containers.
  • Container: Running instance of an image.
  • Dockerfile: Instructions to build an image.
  • Volumes: Persist data outside containers.
  • Networks: Enable communication between containers.

Create a Dockerfile

This Dockerfile sets up a Python 3.11 app, copies project files, installs dependencies, and runs main.py.

# Dockerfile
FROM python:3.11
WORKDIR /app
COPY . .
RUN pip install -r requirements.txt
CMD ["python", "main.py"]

Build and Run

Build the Docker image and run the container:

# Build image
docker build -t my_app .

# Run container interactively, removing it after exit
docker run -it --rm my_app

Docker Compose

Use Docker Compose for multi-container applications. This example exposes port 5000 and mounts local files:

# docker-compose.yml
version: '3'
services:
  web:
    build: .
    ports:
      - "5000:5000"
    volumes:
      - .:/app
    environment:
      - ENV=development

Volumes and Persistence

Keep data outside containers for persistence:

# Create volume
docker volume create my_data

# Run container using volume
docker run -v my_data:/app/data my_app

Workflow Example

Example of building, running, and cleaning up:

cd my_project
docker build -t my_app .
docker run -it --rm -v my_data:/app/data my_app
docker-compose up
docker system prune -f  # Remove unused images and containers

Best Practices

  • Keep images small with multi-stage builds.
  • Pin versions for reproducibility.
  • Use `.dockerignore` to exclude unnecessary files.
