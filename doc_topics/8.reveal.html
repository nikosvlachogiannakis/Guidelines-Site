Creating and Managing Reveal.js Slides

Edit Your Slides

Slides are written in slides.md using Markdown. Use --- to separate horizontal slides, and -- for vertical slides:

# Welcome

---

## Slide 1

Some text

---

- Vertical slide 1

---

- Vertical slide 2

Customizing Theme

Reveal.js themes are in the dist/theme folder. To change the theme, edit index.html and modify this line:

<link rel="stylesheet" href="dist/theme/black.css" id="theme">

You can try other themes like white.css, night.css, moon.css, etc.

Transition Between Slides

Option 1: Set Globally in the Reveal Configuration (index.html)

<script>
  Reveal.initialize({
    transition: 'fade', // or 'slide', 'convex', 'concave', 'zoom', 'none'
    transitionSpeed: 'default' // 'fast', 'slow', or 'default'
  });
</script>

Option 2: Per Slide with data-transition in Markdown

<!-- .slide: data-transition="zoom" -->

Or for vertical slides:

<!-- .slide: data-transition="fade-in slide-out" -->

Change Markdown File

Create a folder called Presentations to store all Markdown files for your presentations. To change the presentation:

  • Open index.html
  • Find: data-markdown="slides.md"
  • Change it to: data-markdown="Presentations/notes.md"

The slides from notes.md will now appear.

Load Slides Locally

Open a terminal inside your folder:

npm install
npm start

Open your browser at http://localhost:8000 to view your slides.

Hosting Slides on GitHub Pages

To publish online for free using GitHub Pages:

  1. Ensure your repo includes: index.html, slides.md, dist/ and plugin/ folders, and an empty .nojekyll file.
  2. Go to GitHub repository → Settings → Pages
  3. Set Source to branch main, folder /, then Save
  4. Wait a few minutes. Your site will appear as: https://your-username.github.io/repo-name/

If the Green Message Does NOT Appear

  • Add an empty .nojekyll file
  • Commit to trigger rebuild: 
    git commit --allow-empty -m "Trigger rebuild"
git push
  • Visit your slides in the browser

Public vs Private Repos and Sharing Slides

Option 1: GitHub Pages (Public Repo)

Slides will be hosted at: https://your-username.github.io/repo-name/. Recommended method.

Option 2: GitHub Pages (Private Repo)

Private repos will not build on GitHub Pages unless on Enterprise plan.

Option 3: Export to PDF

npm install -g decktape
decktape reveal http://localhost:8000 slides.pdf

Managing Large Files with Git LFS

For large media files (.mp4, .pdf), use Git Large File Storage (LFS) instead of committing directly.

Install Git LFS

Ubuntu/Debian:

sudo apt install git-lfs

Windows: Download from git-lfs.com

Set Up Git LFS in Your Repo

git lfs install
git lfs track "*.mp4"
git lfs track "*.pdf"
git add .gitattributes
git add media/my-video.mp4
git commit -m "Add video using Git LFS"
git push

Notes:

  • GitHub LFS free plan: 1 GB storage, 1 GB/month bandwidth
  • View tracked files: git lfs ls-files

Alternatively, upload media externally (Google Drive, web server) and link in slides.
